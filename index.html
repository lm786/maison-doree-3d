About the App
// Maison Dorée — H7006 Web 3D Application

Project Overview

Maison Dorée is an interactive Web 3D application built for the H7006 module. Three café items — a Croissant, a Cup of Coffee, and a Chocolate Chip Cookie — are presented as fully interactive GLB 3D models inside a warm editorial interface. Users can select between models, orbit and zoom in 3D space, toggle wireframe view, control scene lighting, swap model colours, and trigger spin animations — all within a responsive layout.

3D Modelling

All three 3D models are integrated via Three.js's GLTFLoader in GLB format. The GLB format encapsulates geometry, materials, textures, and PBR (Physically Based Rendering) data in a single binary file. Models are rendered with a MeshStandardMaterial pipeline under a three-point lighting rig. Wireframe inspection mode exposes the underlying polygon topology. A colour-swapping system traverses each model's mesh children and updates material colour values in real time using THREE.Color.

Design Rationale

The visual language draws from Parisian café editorial design — warm parchment tones, Playfair Display serif typography, and a dark espresso header anchor the brand identity. CSS custom properties ensure consistent theming. The layout uses CSS Grid for the two-column split, with a responsive breakpoint at 900px. The ticker strip and grain texture overlay add depth. The Three.js canvas occupies the dominant right-hand area, keeping the 3D model as the focal point.

Interactivity & JavaScript Features

Three.js is loaded via ES Modules from the unpkg CDN. The render loop uses requestAnimationFrame for smooth 60fps rendering. OrbitControls allow full drag-to-rotate and scroll-to-zoom interaction. Model switching uses GLTFLoader on demand, clearing the previous model from the scene. Wireframe toggle traverses all mesh children and flips material.wireframe. The spin animation uses timed Y-axis rotation. A real-time colour-swapping system updates THREE.Color on all mesh materials, preserving dark detail meshes via luminance thresholding. An HDR environment map is loaded via RGBELoader using a Venice sunset equirectangular texture enabling physically accurate image-based lighting and realistic reflections on all PBR materials. ACES Filmic tonemapping is applied to the renderer for cinematic colour grading. An ambient café jazz audio track is toggled via the HTML5 Audio API. Two directional lights are independently toggled via buttons.

Accessibility Considerations

All interactive elements include ARIA labels and roles. Model selection cards support keyboard navigation via Tab and Enter. The ticker strip is marked aria-hidden to avoid screen reader noise. Colour contrast meets WCAG 2.1 AA requirements. The prefers-reduced-motion CSS media query pauses the ticker animation for users who require it. Semantic HTML5 elements ensure correct screen reader traversal order.

Testing Strategy

Live testing was conducted across Chrome 120, Firefox 121, and Safari 17 on macOS and iOS. Three.js GLTFLoader performance was profiled using Chrome DevTools — model load times were under 500ms per asset. The responsive layout was validated at 768px, 900px, and 1440px breakpoints. Peer feedback resulted in improved button contrast and addition of keyboard navigation for all model selection cards.
